# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm start        # start the server (node server.js) on http://localhost:3000
npm run dev      # identical — no hot-reload, restart manually after changes
```

No build step, no transpiler, no test suite, no linter configured.

## Architecture

A single-page language flashcard trainer. No framework, no bundler — plain Node.js + Vanilla JS.

```
server.js          Express REST API + static file server
public/
  index.html       SPA shell — all views in one file, toggled via .hide class
  app.js           All client logic (~1100 lines): state, routing, SM-2, CSV parser, sync
  style.css        All styles
data/
  decks/           One JSON file per deck (slug-named, e.g. tds-it.json)
  sync/            One JSON file per sync code (URI-encoded filename)
```

### Server API (`server.js`)

Six routes, all synchronous file I/O with `fs.readFileSync` / `fs.writeFileSync`:

| Method | Path | Purpose |
|--------|------|---------|
| GET | `/api/decks` | List all deck metadata |
| GET | `/api/decks/:id` | Fetch full deck (name, targetLang, cards[]) |
| POST | `/api/decks` | Create deck from parsed CSV cards |
| DELETE | `/api/decks/:id` | Delete deck file |
| GET | `/api/sync?code=` | Load synced progress by code |
| POST | `/api/sync?code=` | Save progress by code |

Deck filenames are generated by `generateSlug()` — lowercased, diacritics stripped, non-alphanumeric → hyphen. Duplicate slugs get a numeric suffix.

### Client State & Routing (`public/app.js`)

Hash-based routing: `#dashboard` renders the deck grid; `#deck/:id` loads the trainer view. All views live in the same HTML, shown/hidden with a `.hide` CSS class.

The global `state` object holds everything:
- `activeDeck` — full deck object from server (includes `cards[]`)
- `progress` — card ID → SM-2 progress object (persisted to `localStorage`)
- `excluded` — Set of card IDs hidden from practice (persisted to `localStorage`)
- `skippedCards` — Set of card IDs skipped this session (reset on deck load)
- `studyAheadMode` — when true, future-due cards are included

### Card Identity

Card IDs are computed at CSV-parse time via `generateStableId()` — a DJB2 hash of the trimmed, lowercased French (`fr`) text. **Changing the French side of a card changes its ID and loses its progress history.**

### Spaced Repetition (`rateCard` in app.js)

SM-2 variant with four ratings. Due dates align to local calendar day boundaries (`endOfToday = 23:59:59.999`). Progress is stored only in `localStorage`; the server only knows about it when the user manually saves via sync code.

### Sync

Progress lives in `localStorage` keyed by `progress_${deckId}` and `excluded_${deckId}`. The sync feature serializes these to the server under a user-chosen plaintext code; anyone with the code can read/overwrite that progress. There is no authentication.

### CSV Format

Three columns: `fr,target,lesson` (lesson is optional — defaults to `"Général"`). Delimiters auto-detected (comma vs semicolon). Quoted cells and escaped quotes (`""`) are supported.
